# Presentation Toolkit

## When to Use

- The user asks for slides, a deck, PPT, PPTX, keynote-style content, or presentation cleanup.
- Existing slides need restructuring, rewriting, visual polish, speaker notes, or export.
- A deck must be verified visually after generation or edits.

## Workflow

1. Clarify the story
   Identify audience, purpose, setting, constraints, and the decision or feeling the deck should drive.
2. Build an outline first
   Define slide roles before visual work: cover, agenda, narrative sections, evidence, demo, risks, close.
3. Reuse theme assets
   Preserve existing masters, fonts, brand colors, slide sizes, and recurring visual patterns.
4. Keep slides scannable
   Use one main point per slide, concise labels, and charts or images only when they support the story.
5. Render and inspect
   Check text overflow, image cropping, chart labels, speaker notes, missing fonts, and final slide count.

## Tool Guidance

- Use structured PPTX libraries or available presentation renderers.
- Avoid editing XML directly unless the required feature is not supported by higher-level tooling.
- Use generated or searched images only when rights and relevance are appropriate.
- When modifying an existing deck, avoid resetting masters or stripping animations/comments unless requested.

## Output Expectations

- Provide the final deck path.
- Summarize slide count and major changes.
- Mention render/export checks and any remaining layout risks.
