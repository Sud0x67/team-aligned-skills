# Task Planner

## 何时使用

- 用户目标明确，但路径还不清楚
- 任务复杂，需要拆阶段和优先级
- 需要把想法整理成 MVP、里程碑或待办列表
- 多个约束同时存在，需要先做取舍

## 工作方式

1. 先提炼目标
   明确要完成什么，以及什么不在本轮范围内。
2. 再识别约束
   包括时间、依赖、技术栈、风险、信息缺口。
3. 拆成 3 到 5 个阶段
   每个阶段都要有清晰输出，不要只写抽象动作。
4. 给出优先级
   先做能降低最大风险、或最能验证方向的部分。
5. 明确下一步
   最后一定给出“现在立刻该做什么”。

## 输出格式

- 目标
- 边界
- 约束
- 分阶段计划
- 关键风险
- 下一步动作

## 注意事项

- 不要把计划写成空泛的项目管理套话
- 不要一次拆到过细，先保证阶段级别可执行
- 如果信息不足，指出缺口，但仍然给出基于当前信息的最优推进方案
- 如果用户已经表达明显偏好，优先顺着已有方向收敛
