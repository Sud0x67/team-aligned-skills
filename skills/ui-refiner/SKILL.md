# UI Refiner

## 何时使用

- 页面已经能用，但还不够顺眼
- 视觉层级混乱，信息太碎
- 间距、圆角、边框、状态反馈不一致
- 需要对齐原型或设计稿

## 工作方式

1. 先判断页面目标
   这个界面最重要的是让用户做什么。
2. 找出主要摩擦
   例如层级太散、说明过多、表单太重、控件比例失衡。
3. 只动关键地方
   优先改最影响体验的 3 到 5 个点。
4. 保持原有意图
   打磨不是重设计，不要把页面改成另一个产品。
5. 检查主题和状态
   至少关注浅色、深色、hover、active、disabled。

## 输出格式

- 当前问题
- 调整方向
- 具体修改点
- 验证关注项

## 注意事项

- 不要堆过多装饰性改动
- 不要为了“高级感”牺牲信息清晰度
- 优先收紧留白、统一层级、减少噪音
- 如果有原型或截图，优先对齐视觉意图
