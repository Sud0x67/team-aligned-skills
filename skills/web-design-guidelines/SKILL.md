# Web Design Guidelines

## When to Use

- The user asks to design, polish, or review a web interface.
- A frontend looks generic, cramped, broken, or visually inconsistent.
- Responsive layout, accessibility, or visual hierarchy needs review.

## Workflow

1. Understand the product category and audience.
2. Reuse the existing design system before inventing new patterns.
3. Make core workflows visible and efficient.
4. Use real or generated visual assets when they clarify the subject.
5. Verify with screenshots across relevant viewports.

## Guardrails

- Do not create a landing page when the user asked for an app or tool.
- Avoid nested cards and decorative clutter.
- Ensure text never overlaps or overflows its controls.
